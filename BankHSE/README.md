

**BankHSE** — это консольное приложение для управления личными финансами. Приложение предоставляет следующие возможности:

- **Операции** над сущностями:
    - **BankAccount**: создание, удаление, просмотр счетов с балансом.
    - **Category**: создание, удаление, просмотр категорий (доходы и расходы).
    - **Operation**: создание операций (тип операции определяется автоматически по выбранной категории), просмотр операций.
- **Автоматическое обновление баланса счета** при добавлении операций.
- **Аналитика**: группировка операций по категориям, подсчет суммарных доходов и расходов и их разницы.
- **Импорт и экспорт данных** в форматах CSV, JSON и YAML.

### Использованные приницпы SOLID GRASP
- **Single Responsibility Principle (SRP)**: Каждый класс отвечает за свою конкретную задачу
- **Open/Closed Principle (OCP)**: Новые функциональности добавляются через наследование без изменения существующего кода (например, добавление новых формтаов экспорта/импорта)
- **Liskov Substitution Principle (LSP)**: Все посетители (CsvVisitor, JsonVisitor, YamlVisitor) могут взаимозаменяемо использоваться как базовый Visitor.
- **Interface Segregation Principle (ISP)**: Разделены интерфейсы ICommand, IAnalyticsStrategy для конкретных задач.
- **Dependency Inversion Principle (DIP)**: Высокоуровневые модули зависят от абстракций (ICommand, IAnalyticsStrategy), а не от конкретных реализаций.
- **High Cohesion и Low Coupling (GRASP)**: Логика разделена по модулям, каждый модуль имеет четкую зону ответственности.

### Паттерны GoF
- **Фабрика (Factory)**: Класс `BankFactory` централизованно создает доменные объекты (BankAccount, Category, Operation), что обеспечивает единообразное создание объектов.
- **Фасад (Facade)**: Фасады `BankAccountFacade`, `CategoryFacade`, `OperationFacade`, `AnalyticsFacade`, `ImportFacade` объединяют логику работы с соответствующими сущностями и упрощают взаимодействие с доменной моделью.
- **Команда (Command)**: Реализованы команды для операций с данными (ExportCommand, ImportCommand и т.д.). 
- **Декоратор (Decorator)**: Класс `TimingDecorator` добавляет функциональность измерения времени выполнения к любым командам без изменения их исходного кода.
- **Шаблонный метод (Template Method)**: Абстрактный класс `DataImporter` задает общий алгоритм импорта данных.
- **Посетитель (Visitor)**: Реализован абстрактный класс `Visitor` и его реализации для экспорта данных в разные форматы (CsvVisitor, JsonVisitor, YamlVisitor).
- **Стратегия (Strategy)**: Интерфейс `IAnalyticsStrategy` и его реализации позволяют динамически выбирать алгоритмы анализа данных. 
- **Наблюдатель (Observer)**: Реализован механизм автоматического обновления баланса счетов при добавлении новых операций. 

### Инструкция по сборке и запуску
Перейдите в папку проекта BankHSE/BankHSE, востановите зависимости командой dotnet restore и соберите проект командой dotnet build. Приложение можно запустить командой dotnet run. Далее следуйте инструкциям на экране.
